# Ubiquitous Language Governance

## Scope
- 本ルールはリポジトリ内の全ディレクトリを対象に適用する。
- 対象には仕様文書、ユーザーストーリー、API関連文書、実装設計メモ、レビューコメントを含む。

## Authoring Rule
- `MUST` 新規作成または更新する文書では `product/ubiquitous/terms.md` を参照し、正式語を使用する。
- `MUST` 用語の意味が曖昧な場合は新規語を先に追加せず、既存語で説明できるかを確認する。
- `SHOULD` 用語の初出では文脈が伝わる文章で記載する。

## Review Rule
- `MUST` すべてのPRでユビキタス言語整合を確認する。
- `MUST` レビュー時に「正式語の使用」「禁止語の混入有無」「定義との意味一致」を確認する。
- `SHOULD` 不整合を見つけた場合は `terms.md` 参照箇所を明示して修正依頼する。

## Approval Rule
- `MUST` `terms.md` への用語追加・変更・削除はPO承認を必須とする。
- `MUST` 変更理由と想定影響範囲をPR説明に記載する。

## Migration Rule
- `MUST` 既存文書は一括置換しない。
- `MUST` 既存文書を更新するタイミングで、該当箇所の用語を正式語へ順次置換する。
- `SHOULD` 大規模改修時は用語置換の対象範囲を事前に明示する。

## Exception Rule
- `MAY` 外部仕様や引用で正式語を使えない場合のみ例外を認める。
- `MUST` 例外時は文書内の `Notes` またはPR説明に、理由・対象範囲・解消方針を記載する。

## Change Process
1. 用語変更案を作成する。
2. `terms.md` の該当エントリを更新する。
3. 影響する文書の更新方針をPRに記載する。
4. PO承認を取得する。
5. マージ後、関連文書を更新時に順次追従させる。
