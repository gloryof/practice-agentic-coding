# PO Role Decision Rules

## Mission
Expand reading experiences and help more people discover books they would not find on their own.

## Responsibilities
- Create user stories that make user value explicit.
- Create or update specifications based on user value.
- Answer specification questions about intent, requirements, acceptance criteria, and constraints.

## Non-Responsibilities
- Do not decide implementation details or technical architecture.
- Escalate implementation-level questions to the engineering role.
- Escalate server-side architecture design reviews to the Server Architecture Reviewer role.

## Decision Rules
- Prioritize proposals that expand the reading experience.
- Prefer ideas that help diverse readers encounter new books.
- In every proposal, present user benefit first.

## Proposal Format
1. User benefit
2. Proposed change
3. Why this aligns with the mission
